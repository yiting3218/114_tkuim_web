# 校園活動報名管理系統

## 一、專案整體說明
本專案為一套校園活動報名管理系統，提供管理者透過瀏覽器介面，
對校園活動報名資料進行 新增、查詢、更新與刪除操作。
系統模擬校園中常見的活動報名情境，例如系上活動.比賽或講座報名，
並示範一個完整的 Web 系統從前端操作到後端資料庫儲存的流程。

## 二、專案主題與目標

### 專案主題
校園活動報名管理系統

### 專案目標
建立一個可實際操作的校園活動報名管理系統
熟悉 RESTful API的設計與 CRUD 操作流程
練習前端（HTML/CSS/JavaScript）與後端（Node.js / Express）之整合
使用 MongoDB 儲存與管理報名資料
模擬完整 Web 系統開發流程（登入-操作-資料同步）

## 三、技術選擇與原因
1. 前端
HTML/CSS/JavaScript
基礎且通用的 Web 技術，適合課程實作與展示
不依賴前端框架，能清楚呈現原生 DOM 操作與資料串接流程
2. 後端
Node.js + Express
輕量、易於建立 RESTful API
適合快速開發與教學用途

MongoDB

Docker
簡化資料庫環境建置，避免系統環境差異問題

## 四、系統架構說明
1. 前端
使用者登入/註冊介面
報名資料輸入與顯示
呼叫後端 API 並顯示結果

2. 後端
提供 RESTful API（Create/Read/Update/Delete）
驗證必要欄位
與 MongoDB 進行資料存取

3. 資料庫
儲存報名者資料（姓名.學號.系級.活動名稱.狀態等）

## 五、功能說明
管理者登入與註冊
新增活動報名資料
顯示所有報名清單
更新報名狀態（待確認/已確認）
刪除報名資料
即時與 MongoDB 同步資料
報名資料統計顯示（總數/已確認/待確認）

## 六、安裝方式與執行指引
1. 環境需求
Node.js
Docker
Git
VS Code

2. 啟動 MongoDB
docker run -d \
  --name mongo-event \
  -p 27017:27017 \
  mongo

3. 啟動後端伺服器
cd backend
npm install
npm run dev
啟動成功後，後端 API 預設運行於: http://localhost:3000

4. 啟動前端畫面
使用 VS Code 的 Live Server
或直接開啟 index.html

## 七、使用指引
1. 開啟前端首頁，進入登入/註冊畫面
2. 註冊帳號或登入後進入管理介面
3. 在 新增報名 頁面填寫以下必填欄位：
姓名 學號 系級 活動名稱
4. 點擊 新增報名 後，資料會送至後端並儲存至 MongoDB
5. 系統會自動切換至 報名清單 頁面
6. 管理者可在清單中：
檢查所有報名資料
更新報名狀態
刪除報名資料
